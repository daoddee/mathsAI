MVP Math Learning App Project Plan

+

A structured blueprint for building a low-cost, high-performance adaptive maths tutoring MVP with AI-driven feedback.

+
+
+
+

1. Recommended Tech Stack

+

The MVP leverages a modern, cost-effective JavaScript stack with free-tier hosting and managed services:

+

Frontend

+
    +
  • Next.js (React + TypeScript) for a single-page experience with SSR/SSG, plus built-in API routes.
    • +
  • Tailwind CSS for rapid, utility-first styling and minimal bundle overhead.
    • +
  • Supplementary libraries like MathJax/KaTeX for rendering equations and Chart.js/Recharts for visualising progress.
    • +
+

Backend

+
    +
  • Next.js API routes running on Node.js/TypeScript to keep frontend and backend unified.
    • +
  • Optional migration path to Express/Railway if separation becomes necessary.
    • +
+

Database & Auth

+
    +
  • Supabase (PostgreSQL) for relational data, auth, and storage with generous free limits.
    • +
  • Supabase Auth supporting email/password and passwordless magic links out-of-the-box.
    • +
+

AI Integration

+
    +
  • Anthropic Claude (Claude Instant for cost efficiency) via the official Node SDK to deliver step-by-step explanations, hints, and feedback.
    • +
+

Hosting & DevOps

+
    +
  • Vercel Hobby tier for deploying the Next.js app and API routes with zero DevOps overhead.
    • +
  • Secrets managed via environment variables both locally (.env.local) and in Vercel.
    • +
  • Monitoring through optional free tools like Vercel Analytics, Sentry, or GitHub Actions.
    • +
+

All components are open-source or have free tiers, supporting a "weekend-to-production" velocity while remaining scalable.

3. REST API Endpoint Plan

+

RESTful route handlers within app/api cover authentication, practice, test, and progress tracking:

+

Auth

+
    +
  • POST /api/auth/register – register users via Supabase Auth.
    • +
  • POST /api/auth/login – issue Supabase JWTs (optional if using Supabase client-side auth).
    • +
+

Practice Mode

+
    +
  • GET /api/practice/next-question?topic=<id> – adaptive question retrieval informed by user history.
    • +
  • POST /api/practice/submit-answer – grade answers, persist attempts, and return Claude-powered explanations.
    • +
+

Test Mode

+
    +
  • POST /api/test/start – create exam sessions, assemble question sets, and return timing.
    • +
  • POST /api/test/submit – evaluate submissions, store results, and provide breakdowns.
    • +
+

Progress & Content

+
    +
  • GET /api/progress – deliver topic-level mastery metrics and recommendations.
    • +
  • GET /api/content/topics and /api/content/questions (optional) – surface taxonomy and seed data.
    • +
+

All protected routes validate Supabase JWTs, ensuring users only access their own records via Row-Level Security.

4. Database Schema (PostgreSQL)

+

Core relational tables support users, content, attempts, and assessments:

+
    +
  • users – Supabase-managed identities (optionally extended with profiles).
    • +
  • topics – hierarchical maths concepts.
    • +
  • questions – question bank with difficulty, options, and canonical answers.
    • +
  • attempts – log of all responses (practice or test) including correctness and timing.
    • +
  • exam_sessions – timed mock exam sessions with aggregated scores.
    • +
  • mastery (optional) – cached proficiency metrics per user/topic for quick lookup.
    • +
+

Sample DDL for key tables:

+ 
CREATE TABLE topics (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  name TEXT NOT NULL,
+  parent_topic_id UUID REFERENCES topics(id),
+  created_at TIMESTAMPTZ DEFAULT now()
+);
+
+CREATE TABLE questions (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  topic_id UUID REFERENCES topics(id),
+  question_text TEXT NOT NULL,
+  answer_options JSONB,
+  correct_answer TEXT NOT NULL,
+  difficulty INT,
+  created_at TIMESTAMPTZ DEFAULT now()
+);
+
+CREATE TABLE attempts (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  user_id UUID REFERENCES auth.users(id),
+  question_id UUID REFERENCES questions(id),
+  exam_id UUID REFERENCES exam_sessions(id),
+  given_answer TEXT,
+  correct BOOLEAN,
+  time_taken INT,
+  answered_at TIMESTAMPTZ DEFAULT now()
+);
+

Indexes on attempts.user_id and questions.topic_id keep adaptive queries efficient.

5. Claude API Integration & Prompt Design

+

Server-side Claude calls power explanations, hints, and feedback:

+
    +
  • Centralised client in lib/aiClient.ts using the official SDK and environment key management.
    • +
  • Claude Instant handles short tutoring responses; larger models reserved for complex reviews.
    • +
  • Prompts encourage Socratic guidance, positive reinforcement, and error analysis.
    • +
  • Optional validation: compare AI-derived answers to canonical solutions before returning to users.
    • +
+

Example prompt:

+ 
You are a supportive maths tutor.
+Question: "x² - 5x + 6 = 0".
+Student answer: "x = 1 or x = 6" (incorrect).
+Explain the correct solution step-by-step, highlight the mistake, and finish with a guiding question.
+

This approach keeps API keys secure, controls usage, and delivers personalised tutoring at minimal cost.

6. Authentication Strategy

+
    +
  • Supabase Auth provides email/password and magic-link flows with minimal setup.
    • +
  • JWT sessions managed by the Supabase client; protected API routes verify tokens server-side.
    • +
  • Email verification, password reset, and optional rate limiting enhance security.
    • +
  • Future role support (students, teachers, admins) achievable via Supabase policies and role flags.
    • +
+

Users enjoy frictionless onboarding while data remains protected through RLS and HTTPS-only access.

7. Core User Workflows

+

Practice Mode

+
    +
  • Topic selection (manual or recommended) informed by mastery gaps.
    • +
  • Adaptive question loop: fetch, answer, receive instant AI feedback, repeat.
    • +
  • Attempts stored for continuous learner-model updates.
    • +
+

Test Mode

+
    +
  • Create timed mock exams with configurable topics or past-paper presets.
    • +
  • Client-side timer with server verification; submissions graded on completion.
    • +
  • Post-exam review includes score breakdowns and optional AI explanations per question.
    • +
+

Review Mode

+
    +
  • Dashboard summarising mastery, recent activity, and past exams.
    • +
  • Drill-down into mistakes, with shortcuts to targeted practice.
    • +
  • Future enhancements: streaks, spaced repetition, teacher oversight.
    • +

8. Initial Seed Content Strategy

+

Focus on Quadratic Equations for launch:

+
    +
  • Create ~20 diverse questions covering factoring, quadratic formula, graphs, and word problems.
    • +
  • Tag each with difficulty levels and optional subtopic metadata.
    • +
  • Source material from self-authored items, DeepMind Mathematics Dataset outputs, or AI-assisted generation with manual verification.
    • +
  • Store hints and solutions for each question to support feedback and hint buttons.
    • +
+

Once validated, expand incrementally to additional GCSE and A-Level topics using the same pipeline.

Deployment steps: connect repo → configure env vars → push → run Supabase migrations → seed content → smoke-test production.
